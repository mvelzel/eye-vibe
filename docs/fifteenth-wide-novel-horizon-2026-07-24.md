# Fifteenth wide horizon: new objects before new parameters

## Purpose

This phase obeys a strict “wide before deep” rule. The preceding method-transfer
portfolio closed six bounded mechanisms but did not explain the Eye Messages.
The response is not to add parameters to the least-bad failure. It is to ask
what other *objects* the same authored data could represent.

“Project-new” below means that the exact construction was absent from this
repository and from targeted read-only searches of `silmä-cryptography` and
`silmä-novel` on 24 July 2026. It cannot prove that nobody has privately
considered the broad idea. No Discord messages, reactions, or calls were made.

## Prior-art and overlap audit

Several tempting descriptions are not new:

- forward occurrence-distance coding was already tested at
  `71/2001 = 0.035482`; a backward tape or distance binning is an extension,
  not a novel object;
- direct Kantele sonification already maps the five eye directions to the five
  notes, tries trigrams as successive notes or chords, tries trigram sums, and
  reports no decoding claim;
- an older Kantele document proposes playing Eye-derived note sequences
  in-game;
- raw-eye adjacent transpositions in `S6`, BWT/MTF, block permutation,
  component selectors, coordinate reassociation, cellular rows, and ordinary
  prefix/phrase grammars all have explicit negative boundaries here;
- a recent cryptography discussion compares Finnish verse variants with
  identical 24-syllable openings and later substitution blocks. That is useful
  source/ancestry motivation, but edit-template and snapshot-delta families
  already exist in the ledger.

Targeted exact searches found:

- no `Huffman`, `"prefix code"`, `"canonical code"`, `"code lengths"`, or
  `Catalan` hit in either Eye channel;
- no `multiset`, `"sorted trigrams"`, `"unordered trigrams"`, or
  `"order type"` hit;
- no `ultrametric`, `"p-adic"`, `"5-adic"`, or `"wreath product"` hit;
- no `"Gray code"`, `Hilbert`, or `Morton` hit;
- no `Sturmian`, `episturmian`, `"Arnoux-Rauzy"`, `"subword complexity"`, or
  `"symbolic dynamics"` hit;
- no `"uniform substitution"` hit. Bare `morphism` has unrelated
  category-theory/isomorphism discussion and is not evidence for a word
  morphism test.

These are novelty-attribution boundaries, not cryptographic evidence.

## Frozen observations that the new ideas must consume

- The renderer alphabet is exactly six typed symbols: five eye directions plus
  authored newline.
- Each first trigram independently unranks to a permutation of those same six
  symbols.
- Each accepted body glyph is an ordered triple in the `5×5×5` eye cube; the
  visible set is exactly raw ranks `0..82`, leaving 42 cube states unused.
- Sorting a triple produces one of 35 size-three multisets over five values.
  The accepted set realizes 31; the only absent classes are
  `333,334,344,444`.
- Seven fixed nonliteral context maps and two independently identified
  families provide held-out structural constraints without plaintext.
- The displayed 26-trigram records, equality-prefix tree, factoradic header
  types, three mod-101 diagonal checks, and game asset chronology remain
  available. None may be silently converted into a fitted parameter.

## First frozen batch

Implement A–D only after this horizon is committed.

### A. Prefix-code family

- Exactly 42 ordered full binary tree shapes with six leaves.
- One tree shape shared by all panels.
- Exactly two header conventions: unranked permutation and inverse.
- Exact renderer tape with the three spatially separated header eyes removed;
  newline remains data.
- Tree and convention selected only on P; bit packing convention and score are
  frozen on a planted fixture before real evaluation.
- Q is read once with the fixed P winner.
- Controls permute the five body eye directions globally while holding headers
  and newline fixed; all 120 permutations are enumerated, and the complete
  tree family is reselected each time.

### B. Multiset/order family

- One canonical combination-with-repetition rank for sorted triples.
- One canonical lexicographic rank among the distinct permutations of that
  multiset.
- No per-panel relabeling or choice of which classes mean letters.
- Remove the known copied openings exactly as in the fourteenth horizon.
- Require a planted 26-symbol multiset channel before using the external
  pattern score.
- Report P and Q separately against complete global permutations of the 83
  accepted labels; no pooled-only promotion.

### C. Five-adic family

- Six component orders only; child-value permutations are implicit in the
  ultrametric and are not fitted.
- Select the order by pairwise LCP preservation on the four first-family
  contexts.
- Test that one order unchanged on the three last-family contexts.
- Report an explicit pairwise contradiction when exact preservation fails.

### D. Gray family

- Standard reflected odd-radix Gray recursion only.
- Six axis orders, eight coordinate reflections, and linear/circular path
  distance: 96 candidates.
- Corresponding adjacent chords only; adding lags after failure is forbidden.
- Select on the first family, test the fixed candidate on the last.
- A planted translation/reflection along the same 125-state path must be
  recovered before the Eye result is admissible.

## Stop and promotion rules

- Failure of A does not license arbitrary code lengths, per-panel trees, or a
  fitted Huffman tree.
- Failure of B does not license an arbitrary 31-symbol substitution or
  occurrence-specific inversion reading.
- Failure of C does not license a fitted hierarchy over the 83 labels; the
  point is the authored three-eye tree.
- Failure of D does not license an optimized space-filling curve; only the
  canonical reflected construction is tested.
- No lane advances on a train-only score, a pleasant rendering, or a
  post-hoc number match.
- If all four fail with passing plants, record four exclusions and return to
  E–H or K. Preserve the unresolved adjacent-only 83-wheel, original trie
  checksum, practice ciphers 3/4, source leads, and Gate clues on the ledger.
