# Twenty-fifth wide horizon — the marker country tag

## Boundary

The universal early-row transfer is closed. Stepping back to the complete
marker layer exposes a new conjunction that was absent from the local ledger
and from a read-only Discord search for `358`.

In canonical engine order, the independently established three natural header
rows have third digits:

```text
0 0 1
1 3 4
2 2 3
```

Their ordinary column sums are:

```text
3 5 8
```

`+358` is Finland's international calling code, as documented by the
[International Telecommunication Union](https://www.itu.int/oth/T0202.aspx?lang=en&parent=T0202).
The same nine digits already have a different, independently constrained
route: the marker edge trail gives BWT last column `300113422`, the LF map
gives the trie-compatible row order, and inverse BWT yields `001123243`, or
orthodox values `(1,38,73)` and ASCII+32 `!Fi`. IANA independently identifies
[`.fi` as Finland's country-code top-level domain](https://www.iana.org/domains/root/db/fi).

This is not a solution. The two readings reuse the same nine third digits,
and the column sum was noticed after the BWT result. Their probabilities may
not be multiplied. At most, the conjunction may disambiguate `Fi` as a
Finland/Finnish tag and show that the first trigrams are deliberately
self-checking metadata.

## Further exact but retrospective consequences

The same grid also gives:

```text

row2 - row1               7

row-2 marker circulation  7 mod 83
358 divmod 83             4,26
```

The last two numbers touch already prominent corpus quantities—four West
panels, 26-glyph displayed records, and the final anchor base position 16.
They are explicitly treated as post-hoc selector candidates, not evidence.
No body transform may be chosen merely because it reproduces one of them.

## First discriminator frozen before enumeration

Lanes A and B are one finite audit and come first. For every distinct
assignment of the fixed multiset `001122334` to the nine fixed header graph
coordinates:

1. reconstruct all nine ranks and retain the existing in-range/distinct
   universe;
2. form the canonical 3×3 third-digit grid;
3. test exact ordered column sums `(3,5,8)`;
4. broaden by all eight D4 views of the grid and test whether any view has
   ordered column sums `(3,5,8)`;
5. read the third digits in the already fixed East-5-first edge trail, run the
   existing LF/BWT decoder, and test exact `!Fi` plus the existing broader
   case-insensitive `Fi` suffix;
6. count exact and broadened joint events;
7. apply the unchanged full factoradic D4/S5/type/coset predicate and report
   which survivors retain the exact ordered country code.

The exact target is diagnostic because it was seen before this freeze. The D4
and case-insensitive versions are the minimum geometric/text broadening, not a
complete correction for every cultural number one might notice. Results are
therefore descriptive even if unique.

No body value, source text, language model, or game asset is consulted in this
first audit.
